## Gaussian integration by parts (IBP): two-track policy

This workspace uses two “Gaussian worlds”, and they should remain **separate entry points** rather
than being forced into a single lemma.

### Track A (analysis/SPDE): Cameron–Martin / covariance-operator IBP on Borel Banach/Hilbert

- **Scope**: Borel probability measures on Banach/Hilbert spaces (SPDE, invariant measures, Gaussian
  analysis).
- **Canonical formulation**: Cameron–Martin measure identity + differentiation → IBP; on Hilbert
  spaces, covariance-operator form.
- **Current source** (upstream queue): `Common/Mathlib/Probability/Distributions/Gaussian/*.lean`.
- **PhysLean exposure**:
  - Once upstreamed to Mathlib, PhysLean should provide only thin “front door” re-export modules.
  - Until upstreamed, PhysLean should *not* depend on `Common/` as a hard requirement.

### Track B (OS/GFF cylindrical): characteristic-functional IBP on `WeakDual` with cylindrical σ-algebra

- **Scope**: distribution-valued fields modeled as `WeakDual` of a test-function space, with the
  cylindrical σ-algebra generated by evaluation maps.
- **Canonical formulation**: differentiate the characteristic functional
  \(\Phi(h) = \int \exp(i\langle \omega, h\rangle)\,d\mu(\omega)\) to obtain IBP identities for
  oscillatory/cylindrical observables (and then polynomial/Wick recursion).
- **Current source**: `gaussian-field-main/GaussianField/Wick.lean`.
- **Stable API surface**:
  - `GaussianField.gaussian_ibp`
  - `GaussianField.gaussian_ibp_general`
  - `GaussianField.hasDerivAt_weighted_exp_leibniz` (differentiate-under-∫ helper)
  - import hub: `gaussian-field-main/GaussianField/IBP.lean`

### Adapter boundary (when connecting the tracks)

If/when a project needs to relate Track A and Track B, the adapter layer should live in a
*separate* module and explicitly state the hypotheses translating between:

- a Gaussian measure on a Borel space (Track A), and
- a cylindrical measure on a `WeakDual` characterized by its characteristic functional (Track B).
